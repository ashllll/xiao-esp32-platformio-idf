# ENS160 air-quality sensor

This guide targets the DFRobot Fermion ENS160 module SEN0515. Confirm the exact module revision before reusing its power or level assumptions.

## What it reports

ENS160 provides an air-quality index, TVOC output, and an **estimated** CO₂-equivalent value derived from gas response. eCO₂ is not direct NDIR or photoacoustic CO₂ measurement and must not be described as such.

The device also exposes validity/status information. Software must distinguish startup, warm-up, normal operation, and invalid output instead of publishing every register value as valid.

## Integration checklist

- Verify module supply and 3.3 V I²C signal compatibility from the module documentation/schematic.
- Use D4/D5 symbols and confirm the configured address for the actual module.
- Follow the datasheet operating-mode and startup sequence.
- Apply temperature/humidity compensation only with correctly encoded, fresh measurements.
- Log status and communication errors with each sample.
- Observe the required warm-up/stabilization period before evaluating trends.

## Acceptance boundary

An address ACK and plausible numbers prove only basic communication. For application claims, compare the complete module against an appropriate reference under documented airflow, temperature, humidity, warm-up, and exposure conditions.

This module is not a certified life-safety, fire, gas-leak, or regulatory CO₂ instrument. See the [primary-source review](../research/ens160-primary-sources.md).
