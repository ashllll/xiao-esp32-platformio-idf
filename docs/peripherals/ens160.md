# DFRobot Fermion ENS160 空气质量传感器

本页整理 DFRobot Fermion ENS160 空气质量传感器（SKU：`SEN0515`）在 XIAO ESP32C3、ESP32S3 和 ESP32C6 上的接入边界。模块可输出 TVOC、eCO₂ 和 UBA AQI，适合室内空气质量趋势、通风和自动化应用。

!!! warning "eCO₂ 不是直接测得的 CO₂"
    ENS160 是金属氧化物多气体传感器，不是 NDIR CO₂ 传感器。`eCO₂` 由 VOC 与氢气响应推算，不能替代需要真实 CO₂ 浓度、法规合规或生命安全用途的仪器。ScioSense 也明确说明 ENS160 不适用于安全关键或生命保护应用。

## 模块与输出

| 项目 | SEN0515 参数 | 说明 |
|---|---:|---|
| 模块供电 | 3.3 V | 只接 XIAO 的 `3V3`，不要接 5 V |
| 标准模式平均电流 | 29 mA | ENS160 芯片典型值 |
| 标准模式启动峰值 | 79 mA，短于 5 ms | 从 IDLE 切换到 STANDARD 时的芯片峰值 |
| 接口 | I²C 或 SPI | 本工程优先使用 I²C |
| 默认 I²C 地址 | `0x53` | SDO/ADDR 拉低后为 `0x52` |
| I²C 速率 | 100 kHz、400 kHz、1 MHz | ESP-IDF/XIAO 工程使用 100 kHz 或 400 kHz |
| TVOC | 0–65,000 ppb，分辨率 1 ppb | 芯片算法输出 |
| eCO₂ | 400–65,000 ppm，分辨率 1 ppm | 等效 CO₂，不是直接 CO₂ 测量 |
| AQI-UBA | 1–5 | 1 最佳，5 最差 |
| 模块尺寸 | 20 × 15 mm | 安装孔距 15 mm、孔径 2 mm |

DFRobot 给出的电气工作范围是 −40–85 °C、5–95 %RH（无凝露）；这只表示器件可工作，并不代表整个范围内都有最佳气体测量性能。ScioSense 推荐在正常室内空气、−5–60 °C、20–80 %RH（无凝露）下使用，并避免长期暴露于侵蚀性或有毒气体环境。

## 接线

最小 I²C 接线只使用模块的 `3V3`、`GND`、`SDA` 和 `SCL`：

| SEN0515 | XIAO 丝印 | 工程宏 | C3 / S3 / C6 GPIO | 说明 |
|---|---|---|---|---|
| `3V3` | 3V3 | — | — | 3.3 V 供电 |
| `GND` | GND | — | — | 必须共地 |
| `SDA` | D4 | `XIAO_I2C_SDA` | 6 / 5 / 22 | I²C 数据 |
| `SCL` | D5 | `XIAO_I2C_SCL` | 7 / 6 / 23 | I²C 时钟 |
| `SDO` | 默认状态 | — | — | 默认地址 `0x53`；接 GND 改为 `0x52` |
| `INT` | 可选空闲 GPIO | — | 按板型分配 | 不使用中断时可不接 |

模块还引出 `SCK`、`SDI`、`SDO` 和 `CS` 供 SPI 使用。选择 SPI 时应按 [SPI 页面](spi.md)重新分配信号和独立 CS，不要同时按 I²C 与 SPI 两套方式驱动模块。

!!! note "DFRobot 示例的引脚不能原样照抄"
    DFRobot Arduino 中断示例里出现了 D4，但本工程已把 XIAO D4 固定为 I²C SDA。需要中断时必须从[引脚资源表](../hardware/pinout.md)选择另一根未占用 GPIO，并针对 C3、S3、C6 及 S3 Sense 扩展板分别检查冲突。

正式设计应核对模块实际版本的上拉电阻。多块 I²C 模块并联时需要计算等效上拉；ESP32 内部上拉只适合短线调试，不应作为正式硬件的唯一上拉。

## 数据含义

| 输出 | 解释 | 使用边界 |
|---|---|---|
| TVOC | 总挥发性有机物的算法浓度，单位 ppb | 适合趋势和空气质量控制，不是对每一种气体的选择性定量 |
| eCO₂ | 根据 VOC 与氢气响应估算的等效 CO₂，单位 ppm | 人员活动与 VOC 来源不符合算法假设时，可能和真实 CO₂ 明显不同 |
| AQI-UBA | 1–5 级室内空气质量指数 | DFRobot 对应为 Excellent、Good、Moderate、Poor、Unhealthy |

读取值时必须同时检查 `DEVICE_STATUS`：

| Validity 值 | 状态 | 处理 |
|---:|---|---|
| 0 | 正常 | 可发布数据，同时保留状态字段 |
| 1 | 预热 | 上电后通常约 3 分钟，不作为稳定值 |
| 2 | 首次启动 | 首次使用的前 1 小时，不作为最终验收值 |
| 3 | 输出无效 | 丢弃数据并记录错误 |

首次启动状态只有在传感器累计完成最初 24 小时连续运行后才会写入非易失存储；在此之前断电，重新上电仍可能再次进入首次启动状态。不要把“预热小于 3 分钟”误写成“新传感器 3 分钟后已完成全部稳定化”。

## 温湿度补偿

ENS160 本身不测量环境温湿度。为提高算法输出的一致性，应从独立温湿度传感器定期更新：

- `TEMP_IN`（`0x13`，小端）：`(摄氏温度 + 273.15) × 64`；
- `RH_IN`（`0x15`，小端）：`相对湿度 %RH × 512`。

两个寄存器都应先写低字节、再写高字节，高字节写入时数据才生效。固定写入 `25 °C / 50 %RH` 只能作为通信演示，不能替代真实环境补偿。湿度补偿默认作用于算法输出，不作用于原始传感器信号。

## ESP-IDF 集成

DFRobot 的 `DFRobot_ENS160` 1.0.1 是面向 Arduino `Wire`/`SPI` 的 MIT 许可库。当前工程使用纯 ESP-IDF 6.0.1，不能直接复制其 Arduino 示例或把头文件加入工程。现阶段不固定未经本工程验证的第三方 ESP-IDF 组件；可按数据手册使用 `driver/i2c_master.h` 实现小型寄存器驱动，不需要额外依赖。

组件应由调用方传入共享的 `i2c_master_bus_handle_t`，自行添加地址为 `0x53`（或已硬件改址的 `0x52`）、最高 400 kHz 的设备句柄，并声明：

```cmake
idf_component_register(
    SRCS "ens160.c"
    INCLUDE_DIRS "include"
    REQUIRES esp_driver_i2c
)
```

最小初始化与采集顺序：

1. 探测 `0x53`；硬件明确把 SDO 接地时改为 `0x52`。
2. 读取 `PART_ID`（`0x00`，小端），必须得到 `0x0160`；错误 ID 立即停止初始化。
3. 写入当前温湿度补偿值。
4. 向 `OPMODE`（`0x10`）写 `0x02`，进入 STANDARD 气体测量模式。
5. 轮询 `DEVICE_STATUS`（`0x20`），先检查错误位、Validity 和新数据位。
6. 从 `0x21` 连续读取 AQI、TVOC 和 eCO₂；TVOC/eCO₂ 都是小端 16 位值。
7. 总线超时或 NACK 采用有限重试；持续失败时移除设备句柄并重新初始化，不能无限阻塞产品任务。

| 寄存器 | 地址 | 长度 | 用途 |
|---|---:|---:|---|
| `PART_ID` | `0x00` | 2 | 期望 `0x60 0x01` |
| `OPMODE` | `0x10` | 1 | `0x00` 深度睡眠、`0x01` 空闲、`0x02` 标准模式、`0xF0` 复位 |
| `TEMP_IN` | `0x13` | 2 | 外部温度补偿 |
| `RH_IN` | `0x15` | 2 | 外部湿度补偿 |
| `DEVICE_STATUS` | `0x20` | 1 | 错误、Validity、新数据标志 |
| `DATA_AQI` | `0x21` | 1 | AQI-UBA 1–5 |
| `DATA_TVOC` | `0x22` | 2 | TVOC，ppb，小端 |
| `DATA_ECO2` | `0x24` | 2 | eCO₂，ppm，小端 |

## 最小验证

本页目前是“官方资料核对 + 三板通用接线设计”，尚未记录 SEN0515 真机实测。接入真实模块后按以下顺序验收：

1. 断电检查 3.3 V、共地、SDA/SCL 方向和上拉电压。
2. 运行 [I²C 地址扫描](i2c.md#i2c-address-scan)，确认只有预期的 `0x53` 或 `0x52`。
3. 读取并记录 `PART_ID=0x0160`。
4. 记录温湿度补偿来源，确认已进入 STANDARD 模式。
5. 连续记录 `DEVICE_STATUS`；Validity 为 1、2、3 时不能标记数据已稳定。
6. 至少观察 TVOC、eCO₂、AQI 和状态 30 分钟；新模块另做首次 1 小时和连续 24 小时运行记录。
7. 对断开 SDA、错误地址和传感器断电分别验证有限重试与恢复行为。
8. 分别编译实际支持的 C3、S3、C6 环境；“能编译”不得写成“传感器真机通过”。

验收记录至少包含板卡型号/变体、SEN0515 模块批次、I²C 地址、供电、上拉、温湿度来源、固件提交、开始时间、Validity 变化和原始输出。测试气氛应遵守数据表关于避免腐蚀性或使传感器中毒气体的限制；一次气味响应只能证明链路有变化，不能证明浓度准确。
