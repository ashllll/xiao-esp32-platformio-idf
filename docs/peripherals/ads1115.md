# ADS1115 + NTC100K B3950 测温

本页提供可编译的 ADS1115 与 NTC100K 使用示例，适用于 XIAO ESP32C3、ESP32S3 和 ESP32C6。示例基于 ESP-IDF 6.0.1 新 I²C 主机 API，使用您指定的 `R25=100 kΩ、Beta=3950 K` 参数。

!!! warning "B3950 是显式配置，不是 NTC100K 的通用属性"
    “NTC100K”通常只说明 25 °C 标称电阻。不同原厂 100 kΩ NTC 的 Beta、端点、公差和 R/T 曲线并不相同。代码中的 3950 K 必须与手中器件的数据表一致；不能套用到任意 100 kΩ NTC。

## 已实现内容

| 路径 | 用途 |
| --- | --- |
| `components/ads1115` | ADS1115 单次单端转换驱动；轮询 OS 就绪位并带超时 |
| `components/ntc_thermistor` | 低侧 NTC 分压反算与 Beta 温度公式 |
| `examples/ads1115_ntc100k` | 三种 XIAO 共用的完整示例工程 |
| `tests/ntc_thermistor_host_test.c` | 25 °C、10 °C 回算和异常边界的主机测试 |

示例没有引入第三方驱动。接入前已检查 Espressif Component Registry 中的 `esp-idf-lib/ads111x`、`bammer187/ads1115` 和 `espp/ads1x15`；为了直接使用 ESP-IDF 6.0.1 新 I²C API并缩小依赖面，本仓库采用独立的小型组件。

## 接线

### ADS1115 到 XIAO

| ADS1115 | XIAO | 说明 |
| --- | --- | --- |
| VDD | 3V3 | ADC 与 GPIO 保持同一 3.3 V 电压域 |
| GND | GND | 共地 |
| SDA | D4 | 代码使用 `XIAO_I2C_SDA`，各板自动映射 |
| SCL | D5 | 代码使用 `XIAO_I2C_SCL`，各板自动映射 |
| ADDR | GND | 7-bit 地址 `0x48` |
| AIN0 | 分压节点 | 测量 NTC 节点电压 |
| AIN1 | 3V3 | 实测分压激励电压，避免把 3.300 V 写死 |
| ALERT/RDY | 不接 | 软件轮询 Config 寄存器 OS 位 |

VDD 旁放置 0.1 µF 去耦。SDA/SCL 必须上拉至 3.3 V；若模块自带上拉，先核对其电压与并联后的等效阻值。

### NTC 分压器

```text
XIAO 3V3 ── 100 kΩ 精密固定电阻 ──+── ADS1115 AIN0
                                    |
                              NTC 100 kΩ B3950
                                    |
                                   GND

XIAO 3V3 ───────────────────────────── ADS1115 AIN1
```

建议在 AIN0 与 GND 间放置约 10 nF 电容，并按实际采样速率验证稳定时间。25 °C 时理想节点约为 1.65 V。

!!! danger "输入范围"
    ADS1115 的 `±4.096 V` 是差分满量程刻度，不是输入耐压。3.3 V 供电时，每个模拟输入仍必须保持在 GND 到 VDD 内。不要给模块 5 V 供电后把 5 V I²C 上拉直接连接 XIAO。

## 参数与换算

示例默认值：地址 `0x48`、100 kHz I²C、AIN0/AIN1 单端、PGA `±4.096 V`（125 µV/LSB）、128 SPS、single-shot、25 ms 超时。AIN0 的起始 Config 字为 `0xC383`。驱动允许配置到 400 kHz，但面包板和未知模块从较保守的 100 kHz 起步。

固定电阻在上、NTC 接地时：

```text
Rntc = Rfixed × Vnode / (Vsupply - Vnode)
T(K) = 1 / (1 / T0(K) + ln(Rntc / R0) / Beta)
T(°C) = T(K) - 273.15
```

配置位于示例的 `main/main.c`：

```c
static const ntc_beta_config_t NTC_CONFIG = {
    .nominal_resistance_ohm = 100000.0,
    .beta_kelvin = 3950.0,
    .nominal_temperature_c = 25.0,
    .fixed_resistor_ohm = 100000.0,
};
```

若 NTC 位于高侧，必须更换电阻反算式。`ln` 是自然对数，公式温度必须使用 Kelvin。

## 编译与运行

只编译，不会烧录设备：

```bash
pio run -d examples/ads1115_ntc100k -e xiao_esp32c3
pio run -d examples/ads1115_ntc100k -e xiao_esp32s3
pio run -d examples/ads1115_ntc100k -e xiao_esp32c6
```

确认串口设备后才执行上传，例如：

```bash
pio run -d examples/ads1115_ntc100k -e xiao_esp32c3 -t upload
pio device monitor -b 115200
```

25 °C 附近的理想日志应接近：

```text
supply=3.300000V node=1.650000V ntc=100000.0Ohm temperature=25.00C
```

实际读数不会精确等于该值。程序会拒绝 `Vnode <= 0`、`Vnode >= Vsupply` 以及非有限结果；这类读数优先排查短路、开路、饱和或接线错误。

## 精度与已知边界

100 kΩ/100 kΩ 分压器在 25 °C 的 Thevenin 源阻抗约 50 kΩ。TI 数据表给出 ADS1115 在 ±4.096 V 档的典型共模输入阻抗为 6 MΩ、差分输入阻抗为 15 MΩ，因此输入负载不能当作零。10 nF 电容有助于抑制噪声，但不能证明已经消除直流负载误差。

误差预算至少包含：

- NTC 的 R25/Beta 公差和单 Beta 模型误差；
- 固定电阻公差与温漂；
- ADS1115 增益、偏移、噪声和输入负载；
- 3.3 V 激励、漏电、线缆和 PCB 污染；
- NTC 自热、安装方式、热梯度和响应时间。

需要更高准确度时，优先使用具体料号的 R/T 表或 Steinhart–Hart 系数，加入低输入偏置缓冲器，并对整机做两点或三点校准。16 位分辨率不等于 16 位系统准确度。

## 真机验收

1. 断电确认上拉电压、ADDR、去耦和分压拓扑。
2. 扫描 I²C，应在 `0x48` 得到 ACK。
3. 用 100 kΩ 精密电阻替代 NTC，核对电阻反算和约 25 °C 结果。
4. 分别用已知电阻覆盖预期温区两端，检查开路/短路行为。
5. 使用可追溯温度参考完成至少两个温度点，并记录原始 ADC code、Vnode、Vsupply 与实际器件参数。
6. 在要支持的每一种 XIAO 板上重复测试。

当前仅完成主机测试与三目标编译；没有 ADS1115/NTC 实物证据，因此不能标记为硬件已验证。
