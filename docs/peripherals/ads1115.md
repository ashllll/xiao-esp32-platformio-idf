# ADS1115 16 位 ADC 实战

本页给出 GY-ADS1115 模块在 XIAO ESP32C3、ESP32S3 和 ESP32C6 上的完整最小接入。实现直接使用 ESP-IDF 6.0 的新 I²C 主机 API，不依赖未经核实的第三方组件。

## 适用范围与来源

| 项目 | 值 |
| --- | --- |
| 芯片 | Texas Instruments ADS1115，16 位 ΔΣ ADC |
| 接口 | I²C，最高 400 kHz |
| 芯片供电范围 | 2.0–5.5 V |
| 本工程推荐供电 | **3.3 V**，保证 SDA/SCL 上拉电平与 XIAO GPIO 兼容 |
| 默认地址 | `0x48`，ADDR 接 GND |
| 官方依据 | [TI ADS1115 数据手册](https://www.ti.com/lit/ds/symlink/ads1115.pdf) |
| 本机当前数据手册 | `.../components/GY_ADS1115/datasheets/ads1115_current_2026-07-18.pdf`，SHA-256 `5fe00cf9509daba7ca311cd0a461debc563dd57598c5de3bf9c31903dc2238da` |

!!! warning "模块供电与逻辑电平"
    ADS1115 芯片允许 5 V 供电，并不代表 XIAO 的 GPIO 可以承受 5 V。常见模块把 SDA/SCL 上拉到 VDD；因此本工程统一给模块供 3.3 V。使用不同模块前，先看原理图并实测上拉电压。

## 接线

三种 XIAO 板在本工程中使用同一组板级别名：

| ADS1115 | XIAO 丝印 | 工程宏 | 说明 |
| --- | --- | --- | --- |
| VDD | 3V3 | — | 3.3 V 供电 |
| GND | GND | — | 共地 |
| SDA | D4 | `XIAO_I2C_SDA` | I²C 数据 |
| SCL | D5 | `XIAO_I2C_SCL` | I²C 时钟 |
| ADDR | GND | — | 地址 `0x48` |
| ALERT/RDY | 可不接 | — | 本例轮询读取 |
| A0 | 被测信号 | — | 单端输入，相对 GND |

模块板通常已经带 I²C 上拉电阻。多块模块共用总线时，应计算并联后的等效阻值，避免上拉过强。不要把超过 VDD 或低于 GND 的电压直接送入模拟输入；测量电池等高电压源时先使用合适的分压和保护电路。

ADDR 可连接 GND、VDD、SDA 或 SCL，对应地址分别为 `0x48`、`0x49`、`0x4A`、`0x4B`。修改地址后同步修改代码中的 `ADS1115_ADDR`。

## 最小驱动

下面的例子执行一次 A0 单端转换：量程 ±4.096 V、128 SPS、单次转换模式。ADS1115 的单端输入仍然不能低于 GND，负数结果应视为接线、噪声或量程配置的排查信号。

```c
#include <stdint.h>

#include "driver/i2c_master.h"
#include "esp_check.h"
#include "esp_log.h"
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "xiao_pins.h"

#define ADS1115_ADDR          0x48
#define ADS1115_REG_CONVERSION 0x00
#define ADS1115_REG_CONFIG     0x01

static const char *TAG = "ads1115";

static esp_err_t ads1115_read_a0(i2c_master_dev_handle_t dev, int16_t *raw)
{
    ESP_RETURN_ON_FALSE(raw != NULL, ESP_ERR_INVALID_ARG, TAG, "raw is null");

    /* OS=1, MUX=A0-GND, PGA=±4.096 V, single-shot, 128 SPS, comparator off. */
    const uint16_t config = 0xC383;
    const uint8_t start[] = {
        ADS1115_REG_CONFIG,
        (uint8_t)(config >> 8),
        (uint8_t)config,
    };
    ESP_RETURN_ON_ERROR(i2c_master_transmit(dev, start, sizeof(start), 100),
                        TAG, "start conversion failed");

    /* 128 SPS 的一次转换约 7.8 ms；留出余量。 */
    vTaskDelay(pdMS_TO_TICKS(10));

    const uint8_t pointer = ADS1115_REG_CONVERSION;
    uint8_t data[2];
    ESP_RETURN_ON_ERROR(
        i2c_master_transmit_receive(dev, &pointer, 1, data, sizeof(data), 100),
        TAG, "read conversion failed");

    *raw = (int16_t)(((uint16_t)data[0] << 8) | data[1]);
    return ESP_OK;
}

void app_main(void)
{
    i2c_master_bus_config_t bus_cfg = {
        .i2c_port = 0,
        .sda_io_num = XIAO_I2C_SDA,
        .scl_io_num = XIAO_I2C_SCL,
        .clk_source = I2C_CLK_SRC_DEFAULT,
        .glitch_ignore_cnt = 7,
        .flags.enable_internal_pullup = true,
    };
    i2c_master_bus_handle_t bus;
    ESP_ERROR_CHECK(i2c_new_master_bus(&bus_cfg, &bus));

    i2c_device_config_t dev_cfg = {
        .dev_addr_length = I2C_ADDR_BIT_LEN_7,
        .device_address = ADS1115_ADDR,
        .scl_speed_hz = 400000,
    };
    i2c_master_dev_handle_t ads1115;
    ESP_ERROR_CHECK(i2c_master_bus_add_device(bus, &dev_cfg, &ads1115));

    while (true) {
        int16_t raw;
        if (ads1115_read_a0(ads1115, &raw) == ESP_OK) {
            const float volts = (float)raw * 4.096f / 32768.0f;
            ESP_LOGI(TAG, "A0 raw=%d voltage=%.6f V", raw, (double)volts);
        }
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
```

## 最小验证

1. 断电检查 VDD、GND、SDA、SCL 和 ADDR 接线。
2. 先运行 [I²C 地址扫描](i2c.md#i2c-address-scan)，应在 `0x48` 得到 `ESP_OK`。
3. A0 接 GND，原始值应接近 0；少量正负抖动属于噪声，不能据此宣称精度达标。
4. A0 接一个低于 3.3 V 的已知稳定电压，用校准过的万用表作参考。
5. 记录板型、供电、PGA 量程、采样率、参考表读数、原始码和换算值。
6. 分别对实际要支持的 `xiao_esp32c3`、`xiao_esp32s3`、`xiao_esp32c6` 环境执行编译。

## 常见问题

| 现象 | 优先检查 |
| --- | --- |
| 探测不到 `0x48` | ADDR 状态、SDA/SCL 是否接反、共地、模块是否有上拉 |
| 总线一直忙或超时 | 上拉等效阻值、线长、供电不稳、错误电平、频率过高 |
| 读数总是满量程 | 输入超过所选 PGA 满量程，或输入超过芯片供电范围 |
| 读数比例固定偏差 | 分压比、PGA 配置、换算常数、万用表参考值 |
| 读数跳动明显 | 输入悬空、布线靠近开关节点、电源去耦、采样率和数字滤波 |

不要把旧归档文档中出现的包名直接加入 `lib_deps`。先在 ESP-IDF Component Registry 或包的官方仓库确认其真实存在、维护状态、许可证和 ESP-IDF 6.0 兼容性；本页示例不需要该依赖。
