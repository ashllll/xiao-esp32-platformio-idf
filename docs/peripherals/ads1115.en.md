# ADS1115 practical guide

ADS1115 is a 16-bit delta-sigma ADC with an I²C interface, programmable gain, multiple input selections, and a comparator function. Resolution in bits does not equal absolute accuracy.

## Wiring and range

Use a 3.3 V-safe module and confirm its pull-up rail. The analog inputs must stay within the absolute and common-mode limits in the TI datasheet; selecting a narrow full-scale range does not permit input beyond the supply rails.

Record:

- module schematic and address-pin state;
- single-ended or differential input selection;
- programmable-gain/full-scale setting;
- data rate and conversion mode;
- source impedance and input filtering;
- ALERT/RDY use, if any.

## Conversion flow

Write a complete configuration, wait for conversion readiness using a bounded timeout, read the signed conversion code, and scale it with the selected full-scale range. Reject configuration mismatches rather than applying the wrong scale.

## Practical OPT101 pairing

When ADS1115 reads an OPT101 module, first prove that the optical sensor output stays within both devices' electrical limits. Characterize zero offset, gain, noise, saturation, and reference-light response. The resulting value is not lux until the whole optical chain is calibrated.

## Acceptance

Apply known voltages within range, test positive/negative differential codes where applicable, compare data rates and noise, and verify behavior at open input and saturation. Cite the exact [TI ADS1115 datasheet](https://www.ti.com/product/ADS1115) used for the design.
