# ADS1115 + NTC100K B3950 temperature sensing

This page provides a buildable ADS1115 and NTC100K example for the XIAO ESP32C3, ESP32S3, and ESP32C6. It uses the ESP-IDF 6.0.1 I²C master API and the user-selected `R25=100 kΩ, Beta=3950 K` parameters.

!!! warning "B3950 is explicit configuration, not a universal NTC100K property"
    “NTC100K” normally identifies only the nominal resistance at 25 °C. Beta, its endpoints, tolerances, and R/T curves differ between parts. Confirm 3950 K against the exact thermistor datasheet.

## Included implementation

| Path | Purpose |
| --- | --- |
| `components/ads1115` | Bounded single-shot ADS1115 driver that polls the OS-ready bit |
| `components/ntc_thermistor` | Low-side divider resistance and Beta-equation conversion |
| `examples/ads1115_ntc100k` | Complete example shared by all three XIAO targets |
| `tests/ntc_thermistor_host_test.c` | Host tests for 25 °C, a 10 °C round trip, and invalid boundaries |

The example has no third-party driver dependency. We reviewed `esp-idf-lib/ads111x`, `bammer187/ads1115`, and `espp/ads1x15` in the Espressif Component Registry, then kept a small local component to use the ESP-IDF 6.0.1 I²C API directly.

## Wiring

| ADS1115 | XIAO | Purpose |
| --- | --- | --- |
| VDD | 3V3 | Keep ADC and GPIO in the same 3.3 V domain |
| GND | GND | Common ground |
| SDA | D4 | `XIAO_I2C_SDA` maps each board |
| SCL | D5 | `XIAO_I2C_SCL` maps each board |
| ADDR | GND | 7-bit address `0x48` |
| AIN0 | Divider node | Measure the NTC divider |
| AIN1 | 3V3 | Measure actual excitation instead of assuming exactly 3.300 V |
| ALERT/RDY | Unconnected | Software polls the Config OS bit |

Place 0.1 µF close to ADS1115 VDD. Pull SDA/SCL up only to 3.3 V and inspect breakout-board pull-ups before connecting it.

```text
XIAO 3V3 ── 100 kΩ precision fixed resistor ──+── ADS1115 AIN0
                                               |
                                         NTC 100 kΩ B3950
                                               |
                                              GND

XIAO 3V3 ──────────────────────────────────────── ADS1115 AIN1
```

An approximately 10 nF capacitor from AIN0 to ground is a useful starting point; verify its settling behavior at the actual sampling rate. The ideal 25 °C node voltage is about 1.65 V.

!!! danger "Input range"
    `±4.096 V` is a differential full-scale setting, not input tolerance. With a 3.3 V supply, each analog input must remain between GND and VDD. Never expose XIAO GPIO to 5 V breakout pull-ups.

## Configuration and equations

Defaults are address `0x48`, 100 kHz I²C, single-ended AIN0/AIN1, ±4.096 V PGA (125 µV/LSB), 128 SPS, single-shot mode, and a 25 ms timeout. The AIN0 start word is `0xC383`. The driver accepts up to 400 kHz, but breadboards and unknown breakouts start at the more conservative 100 kHz.

For a high-side fixed resistor and low-side NTC:

```text
Rntc = Rfixed × Vnode / (Vsupply - Vnode)
T(K) = 1 / (1 / T0(K) + ln(Rntc / R0) / Beta)
T(°C) = T(K) - 273.15
```

The example makes every thermistor assumption visible:

```c
static const ntc_beta_config_t NTC_CONFIG = {
    .nominal_resistance_ohm = 100000.0,
    .beta_kelvin = 3950.0,
    .nominal_temperature_c = 25.0,
    .fixed_resistor_ohm = 100000.0,
};
```

Use the natural logarithm and Kelvin in the equation. Reverse the resistance equation if the NTC is on the high side.

## Build and run

These commands compile only; they do not flash hardware:

```bash
pio run -d examples/ads1115_ntc100k -e xiao_esp32c3
pio run -d examples/ads1115_ntc100k -e xiao_esp32s3
pio run -d examples/ads1115_ntc100k -e xiao_esp32c6
```

Upload only after identifying the intended serial device:

```bash
pio run -d examples/ads1115_ntc100k -e xiao_esp32c3 -t upload
pio device monitor -b 115200
```

An ideal 25 °C result is approximately:

```text
supply=3.300000V node=1.650000V ntc=100000.0Ohm temperature=25.00C
```

The component rejects `Vnode <= 0`, `Vnode >= Vsupply`, and non-finite results. Treat these as likely short, open, saturation, or wiring faults.

## Accuracy limits

The 100 kΩ/100 kΩ divider has about 50 kΩ Thevenin resistance at 25 °C. TI specifies typical common-mode and differential input impedances of 6 MΩ and 15 MΩ at the ±4.096 V setting, so loading is not zero. A capacitor can reduce noise but does not prove DC loading error is gone.

The error budget must include thermistor R25/Beta tolerance and model error, fixed-resistor tolerance and drift, ADS1115 gain/offset/noise/loading, excitation and leakage, self-heating, thermal gradients, and response time. For higher accuracy, use the exact part's R/T table or Steinhart–Hart coefficients, consider a low-bias buffer, and calibrate the assembled system at two or three points. Sixteen-bit resolution is not sixteen-bit system accuracy.

## Hardware acceptance

1. With power removed, verify pull-up voltage, ADDR, decoupling, and divider topology.
2. Scan I²C and confirm an ACK at `0x48`.
3. Replace the NTC with a precision 100 kΩ resistor and verify resistance and approximately 25 °C.
4. Use known resistors at both ends of the intended range and test open/short handling.
5. Validate at two or more temperatures and record raw ADC code, Vnode, Vsupply, and exact part data.
6. Repeat on every XIAO target that will be supported.

Only host tests and target compilation are currently established; no physical ADS1115/NTC setup has been validated.
