# OPT101 光照传感器模块

## 器件与模块边界

OPT101 是 Texas Instruments 的单片光电二极管与跨阻放大器，输出电压随入射光功率增加。它是**模拟光功率传感器**，不会直接输出 lux，也没有 I²C 地址。将电压换算为照度需要考虑光源光谱、入射角、光学结构和系统标定；只套用一个网上公式不能得到可追溯的 lux。

“优信电子 OPT101 模块”是板级商品，不等同于裸芯片数据表。未取得该批次的正式原理图前，板载反馈网络、滤波、输出端保护和引脚排列都属于待核实项。下面只采用 TI 已确认的芯片特性，并按 XIAO ADC 的安全范围设计接入。

## 已确认的裸芯片参数

| 项目 | OPT101 |
|---|---|
| 单电源范围 | 2.7–36 V |
| 静态电流 | 典型 120 µA |
| 内部反馈电阻 | 1 MΩ |
| 650 nm 光响应度 | 典型 0.45 A/W |
| 内部 1 MΩ 下的带宽 | 典型 14 kHz |
| 电气特性规定温度范围 | 0–70 °C |

这些数值描述 OPT101 裸芯片，不能用来声称某块模块的输入电压范围或 lux 精度。虽然芯片本身可用高电压供电，XIAO ADC 绝不能接收超过 GPIO 允许范围的模拟电压。本项目优先让模块与 XIAO 同用 3.3 V；如果模块必须用更高电压，必须先测量最大输出并设计外部分压/钳位和源阻抗，不能直接连接。

## 接线

| 模块信号 | XIAO | GPIO（C3 / S3 / C6） | 说明 |
|---|---|---:|---|
| `VCC` | `3V3` | — | 仅在确认模块支持 3.3 V 后连接 |
| `GND` | `GND` | — | 模拟地共地，走线远离开关噪声 |
| `OUT`/`AO` | D1 | 3 / 2 / 1 | `XIAO_D1`，作为避开 C3 D0 启动配置脚的候选；各板 ADC unit/channel 仍须确认 |

不要按表格盲接：先核对模块丝印、实物走线和商家原理图，并用万用表确认暗处与强光下的输出范围。这里选 D1 是为了避免让 OPT101 的主动模拟输出连接 ESP32C3 的 D0/GPIO2 启动配置脚，并不等于三块板已经完成真机验证；切换板卡时仍要核对该 SoC 的 ADC unit/channel、衰减有效范围和复位期间限制。

## ESP-IDF 采集

使用 ESP-IDF 的 ADC oneshot 驱动和校准驱动：

1. 为当前板型把 `XIAO_D1` 映射到明确的 ADC unit/channel；编译期拒绝未知目标。
2. 选择覆盖实测最大输出的最低衰减档，不要把芯片供电电压当作 ADC 可测上限。
3. 多次采样并采用均值/中位数；根据噪声评估是否增加靠近 ADC 引脚的电容。
4. 使用目标支持的校准方案，把 raw code 转成 mV；原始码不能跨不同 ESP32 或衰减设置直接比较。
5. 保存暗电压、参考光源条件、温度、几何位置和校准系数。

组件应依赖 `esp_adc`，使用 `esp_adc/adc_oneshot.h`、`esp_adc/adc_cali.h` 和 `esp_adc/adc_cali_scheme.h`。不同 XIAO 型号的 ADC 非线性、衰减范围和校准方案并不完全相同，因此驱动 API 可共享，通道与校准配置必须按目标区分。

## 从电压到可用数据

建议分三层保存：

- `adc_raw`：仅用于诊断当前板卡和配置。
- `output_mv`：经过 ESP-IDF ADC 校准的模块输出电压。
- `relative_light` 或经实验标定的工程单位：包含校准版本和测试条件。

TI 给出的 650 nm 响应度是单一波长的典型值，不等于人眼 V(λ) 加权的照度响应。若需要 lux，应在目标光源、扩散罩、距离和角度条件下与可溯源照度计做多点标定，并报告误差范围；更换 LED 光谱或机械结构后重新标定。

## 最小验证

- 上电前确认模块输出不会超过所选 XIAO ADC 的有效输入范围。
- 遮光记录暗电压，再以固定位置逐级改变光强，确认电压单调且未饱和。
- 用万用表与校准后的 ADC mV 对读，评估偏差、噪声和供电纹波。
- 对每块目标板分别验证，不能把 S3 的 ADC 校准结果直接复制到 C3/C6。
- 若要发布 lux，完成至少多个光强点和目标光谱的标定，并保存参考仪器信息。
