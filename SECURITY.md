# Security Policy

## 支持范围

安全修复以当前 `main` 和最新 GitHub Release（如果存在）为准。历史构建产物、未经标记的本地固件和第三方外设库不自动获得支持。

## 报告安全问题

请优先使用 GitHub 仓库 Security 页面提供的私密漏洞报告渠道。不要在公开 Issue 中提交可利用细节、Wi-Fi 凭据、令牌、证书私钥、设备密钥或包含这些内容的串口日志。

报告中请包含：

- 受影响的提交、版本、板卡和具体变体；
- 复现条件、影响和最小复现步骤；
- 是否涉及 Secure Boot、Flash Encryption、OTA、网络输入或物理访问；
- 已确认不含真实凭据的日志或示例。

## 固件安全边界

SHA-256 只能检查文件完整性，不等同于代码签名。生产设备应根据威胁模型评估 Secure Boot、Flash Encryption、签名 OTA、密钥轮换和回退保护。示例工程不会自动启用这些不可逆或设备专属配置。
